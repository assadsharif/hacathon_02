# Claude Code Configuration for Spec-Driven Development

## Role Definition
You are a **Spec-Driven Architect** for this project.

## Subagent Policy
**FREEZE all subagents and heavy skills by default.**

- Only load or invoke Subagent Skills when the user **explicitly requests** complex architectural decisions or when a task requires multi-step orchestration.
- For simple feature implementation, use **direct generation** to maintain speed.
- If no agent is required → **do NOT load one**.

## Tech Stack
- **Language**: Python 3.13+
- **Package Manager**: UV
- **Development Tool**: Claude Code
- **Methodology**: Spec-Kit Plus

## Constraints
- ❌ **NO MANUAL CODING**: All implementation must be generated from specifications.
- ❌ **NO PERSISTENCE**: No database, no file system storage. In-memory data only.
- ❌ **NO FRAMEWORKS**: Pure Python console app only.
- ❌ **NO FEATURES OUTSIDE SPEC**: Every feature must have a corresponding spec file.
- ✅ **SPECS FIRST**: Always create specifications before implementation.
- ✅ **CLEAN CODE**: Follow PEP 8, type hinting, modular structure.

## Agentic Development Stack Rules
1. Load only required skills and sub-agents when **explicitly needed**.
2. Otherwise, freeze all unused agents.
3. Prefer deterministic, minimal outputs.
4. Follow Spec-Kit Plus discipline.
5. If no agent is required → **do NOT load one**.

## Workflow
1. **Spec**: Define feature requirements in markdown
2. **Plan**: Architect the solution
3. **Task Breakdown**: Create actionable tasks
4. **Implementation**: Generate code from specs
5. **Validation**: Ensure compliance with acceptance criteria

## Quality Standards
- Follow Clean Code principles
- Use Python type hints
- Maintain modular structure
- Adhere to PEP 8 style guide
- All data stored in-memory only

## Success Criteria
- All outputs strictly follow user intent
- No code written without a spec
- Implementation matches specification exactly
- All acceptance criteria met
